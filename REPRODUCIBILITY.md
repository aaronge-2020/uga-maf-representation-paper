# Reproducibility

This repository is a public, curated reproduction bundle for the UGA/MAF representation manuscript.
It preserves the `projects/cgr_validation` reproduction layout: run the manuscript workflow from `uga_maf_representation_paper/`.

## Setup

```bash
conda env create -f uga_maf_representation_paper/environment/environment.yml
conda activate uga-maf
```

On this workstation the tested Python executable was `D:\Programs\Miniconda3\python.exe`.

## One-command reproduction

```bash
cd uga_maf_representation_paper
python src/run_all_experiments.py --config config/experiment_settings.fresh_manuscript.yaml --paths config/paths.yaml
```

The paths file uses `${repro_root}`, so the command expects this repository root to contain `uga_maf_representation_paper/`, `data/GRCH37/`, and the curated `cgr_validation_results/` input subtrees.

The GRCh37 reference is stored in Git LFS as `data/GRCH37/GCF_000001405.13/GCF_000001405.13_GRCh37_genomic.fna.gz` to stay below GitHub's per-file LFS limit. The first non-dry-run execution automatically expands it to the indexed `.fna` path expected by the feature builders; dry-run mode reports the compressed reference as available without expanding it.

## Dry run

```bash
cd uga_maf_representation_paper
python src/run_all_experiments.py --config config/experiment_settings.fresh_manuscript.yaml --paths config/paths.yaml --dry-run
```

## Regenerate figures and manuscript tables only

```bash
cd uga_maf_representation_paper
python src/utils/make_all_figures.py --config config/experiment_settings.fresh_manuscript.yaml --paths config/paths.yaml --strict
```

## Intentional cache rebuild

```bash
cd uga_maf_representation_paper
python src/run_all_experiments.py --config config/experiment_settings.fresh_manuscript.yaml --paths config/paths.yaml --refresh-cache
```

## Included outputs

Final manuscript outputs are under `uga_maf_representation_paper/results/manuscript/`.
Normalized benchmark outputs are under `uga_maf_representation_paper/results/tables/`.

Feature caches, checkpoints, and `results/work/` are intentionally excluded from GitHub to minimize Git LFS storage. They are regenerated by the workflow.
