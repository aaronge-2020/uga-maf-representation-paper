# EXP025 Payload Absence Benchmark

## Inputs

- UGA payload encoders in `uga_atlas` and `vdkm`.
- Simulated indel-like payload states and patient mixtures generated by the EXP025 runner.

## Construction

1. Compare legacy zero padding, masked payload encoding, and length-coded payload encoding.
2. Enumerate payload-state collisions under each encoding.
3. Simulate exposure recovery for indel-like mixtures.
4. Summarize collision counts, recovery error, process recovery accuracy, geometry, and dimensional efficiency.

## Dimensionality

Dimensionality is determined by the active UGA model's `d_context`, `d_payload`, and payload schema. Masked payloads use 3 bits per payload slot; length-coded payloads use nucleotide bits plus a compact length code.

## Preserved, Added, Lost

- Preserved: REF/ALT payload sequence identity within the selected payload depth.
- Added: explicit absence or length information to distinguish true bases from missing payload slots.
- Lost or smoothed: payload sequence beyond the selected truncation depth.

## Leakage Controls

This is a simulated encoding benchmark with no patient labels. It evaluates representational collisions and recovery behavior under fixed simulation parameters.
